# /check-backlog — ревизия беклога (bugs/ и plans/) и простановка DONE

Со временем в `bugs/` и `plans/` накапливаются файлы, которые завершены (DONE), но не помечены тегом `DONE` — и
листинг файлов больше не показывает, что осталось. Этот навык наводит порядок: собирает текущий список открытых И
проставляет тег DONE действительно закрытым файлам.

Опирается на соглашение о теге `DONE` в имени файла (см. `AGENT_GUIDE.md` → «Беклог и тег DONE»).

## Что делать

1. **Собери все файлы беклога:**
   - `ls bugs/` — все документы по багам.
   - `ls plans/` и `ls ideas/` — идеи/фичи и ворклоги/планы.
   - ⚠️ `plans/` содержит СПРАВОЧНЫЕ документы (не задачи): `MASTER_PLAN.md`, `PROJECT_STRUCTURE_EXTERNAL_MAP.md`, `GOAL.md`,
     `context.md` и т. д. — НЕ помечай их тегом DONE; это живые справочники, а не закрываемые задачи. Помечай только
     конкретные баги/идеи/фичи/задачи.

2. **Раздели на два списка по имени файла:**
   - **Открытые (без `DONE` в имени)** — кандидаты в беклог «реализовать». Отметь их.
   - **Помеченные `DONE`** — уже закрыты, пропусти.

3. **Для КАЖДОГО открытого файла — определи реальный статус** (не угадывай!):
   - Прочитай документ: есть ли внутри статус ✅/CLOSED/IMPLEMENTED/FIXED/PROVEN?
   - Сверься со `STATUS.md` и, при необходимости, с кодом/`git log` — действительно ли это сделано?
   - **Если действительно DONE и подтверждено** → выполни простановку DONE (шаг 4).
   - **Если НЕ сделано / частично / только исследование (🔧/🟡/🔬)** → оставь открытым, добавь в текущий беклог (с
     короткой пометкой «что осталось»).
   - **Если неясно** → НЕ помечай; оставь открытым и отметь «требует проверки».

4. **Простановка DONE (только для подтверждённо закрытых):**
   - Переименуй, вставив `DONE` после номера, сохранив историю:
     `git mv bugs/13_detach_crash.md bugs/13_DONE_detach_crash.md` (не меняй номер; формат
     `<NN>_DONE_<name>.md`).
   - **Допиши раздел статуса внутри документа**, например:
     ```
     ## ✅ STATUS: DONE (<date>)
     What was done: <короткая сводка исправления/реализации>.
     How verified: <сборка/стенд/измерение/хеш коммита/итерация цикла>.
     ```
   - Закоммить (в цикле — по обычной дисциплине коммитов; иначе пакетом в конце ревизии).

5. **Сформируй сводку:**
   - Короткий список ОТКРЫТЫХ задач (беклог) с приоритетом: доделать начатое > баги > новые идеи.
   - Список файлов, помеченных `DONE` в этой ревизии.
   - Всё, что осталось как «требует проверки» (неясный статус).

## Заметки
- Аккуратная ревизия: помечай DONE только то, что РЕАЛЬНО закрыто и проверено — лучше оставить открытым, чем
  ошибочно закрыть.
- Справочные документы в `plans/` никогда не помечаются тегом DONE.
- В автоциклах и при `/refresh-context` вызывай этот навык раз в несколько итераций, не каждую итерацию.
- Если всплывают вопросы уровня человека — заведи в `interviews/` + отметь `STATUS.md`, не решай вслепую.
