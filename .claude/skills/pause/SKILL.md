# /pause — завершить рабочую сессию

Владелец делает паузу. Выполни процедуру завершения **по порядку**. Коротко проговаривай каждый шаг в чате.
Не пропускай шаги. Если шаг падает — остановись, скажи владельцу, не продолжай вслепую.

## Шаг 1. Записать статус и планы в STATUS.md

Обнови `STATUS.md`:
- **Что было сделано за эту сессию** — конкретно, с привязкой к багам/фичам и файлам.
- **Текущая позиция** — что работает, что в процессе, где мы находимся.
- **Планы на следующую сессию** — чёткий чеклист «где продолжить» (команды, файлы, что проверить
  первым делом). Помни: следующая сессия агента начинается с пустым контекстом — пиши так, чтобы можно было сразу включиться.
- Переведи относительные даты в абсолютные (узнай сегодняшнюю дату из контекста / `date`).

Сверься с активными документами по багам в `bugs/` и отрази их статус.

## Шаг 2. Обновить README

Приведи `README.md` в соответствие с реальностью: статус фаз, рабочие фичи, инструкции. Если README
двуязычный, держи обе версии синхронными. Не выдумывай — отражай только то, что действительно сделано и проверено.

## Шаг 3. (Пере)собрать / перегенерировать артефакты

`<Запусти сборку проекта и любую перегенерацию артефактов (например, отрендеренного README.pdf). Для
собственного проекта этого фреймворка: `node tools/build-framework.mjs` перегенерирует KAIF.md, а `node tools/readme-pdf.mjs`
перегенерирует README.pdf.>` Если сборка падает, остановись и покажи ошибки — не коммить сломанное состояние.

## Шаг 4. Закоммитить и запушить

`<Используй свой инструмент/процесс коммита. Если он есть (например, tools/commit.mjs, который поднимает номер сборки, добавляет, коммитит,
пушит), запусти его. Иначе: git add -A && git commit -m "..." && git push.>`

Стиль сообщения (из `AGENT_GUIDE.md`): `feat:` / `fix:` / `docs:` / `refactor:` / `ci:` + одна строка.
Заверши сообщение строкой:
```
Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>
```

## Шаг 5. Отчёт

Кратко отчитайся владельцу: что записано, что собрано, хеш коммита, что запушено, и
одна строка — главный план на следующую сессию. На этом пауза завершена.

## Заметки

- Если пуш отклонён (non-fast-forward) — `git pull --rebase`, повтори пуш, затем расскажи владельцу
  о расхождении.
- Сгенерированные артефакты, попадающие под gitignore (например, README.pdf, результаты сборки), не будут закоммичены — это нормально.
