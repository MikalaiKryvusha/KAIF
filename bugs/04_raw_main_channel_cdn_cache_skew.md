# Bug 04 — sha256 mismatch артефактов раздачи: CRLF внутри бандла vs git-нормализация блоба

**Status:** ✅ DONE (диагноз уточнялся дважды — история ниже сохранена честно)
**Version/build:** dist d3a8f1e → фикс 2026-07-16 · **When/context:** полевой тест ДЗ-02
(homeworks/02), слабая ЛЛМ в Zoo Code.

## Symptom

Загрузчик падает `sha256 mismatch for KAIF-CORE-BUNDLE.md` на `--channel main` **и на закреплённом на
коммит `--source`** (второй факт — от слабой модели: она перепроверила и не поверила первой теории —
и была права).

## История диагноза (важно как урок)

1. **Первая теория — «кэш CDN рассинхронизирован»** (правдоподобно: манифест/ядро на raw совпадали,
   бандл — нет). Дала неверный прогноз: пин на коммит должен был помочь — **не помог**.
2. **Контрфакт от поля** (пин иммутабелен, а mismatch тот же, hash идентичный) убил теорию кэша.
3. **Измерение вместо теории:** `git ls-files --eol` → рабочий бандл `w/mixed` (408356 байт, ~2811
   CRLF), блоб `i/lf` (405545 байт). Разница — ровно нормализация CRLF→LF при коммите.

## Root cause (настоящий)

Сборка встраивает в бандл байты рабочей копии; часть framework-файлов лежала с CRLF (наследие
autocrlf-чекаута — файлы, которые не переписывались инструментами с LF). Манифест считал sha256 по этим
байтам (с CRLF). `.gitattributes` (`* text=auto eol=lf`) нормализует текст при коммите → **блоб ≠
рабочей копии** → raw.githubusercontent (отдаёт блоб) ≠ манифесту. Расхождение детерминированное и
вечное — на любом ref. CDN был ни при чём.

## Fix

1. **Сборка нормализует всё в LF** (`\r\n → \n` при чтении каждого источника в `build-framework.mjs`):
   байты артефактов = блоб = raw = sha в манифесте, независимо от состояния рабочей копии.
2. **`.gitattributes`: `dist/** -text`** — git никогда не трансформирует артефакты раздачи (страховка
   класса).
3. Подсказка в ошибке загрузчика (из первой итерации) остаётся — она полезна и для сетевых сбоев;
   формулировка про кэш смягчена реальностью: сначала повтори, затем пин, никогда — обход.

## Верификация

- Пересборка: бандл `i/lf w/lf`, размер == блобу, sha файла == манифесту == sha блоба.
- После пуша: raw по закреплённому новому коммиту отдаёт байты, совпадающие с манифестом (проверено
  скачиванием и хешированием).
- Поле: слабая модель завершила загрузку с новым пином (см. homeworks/02, результаты владельца).

## Дополнение 2026-07-17 — кэш ассетов release-канала после `--clobber`

При перезаливке ассетов в СУЩЕСТВУЮЩИЙ тег (`gh release upload v1.5 … --clobber`)
`releases/latest/download/...` ещё несколько минут отдавал ПРЕЖНИЕ байты: смок скачал ядро 42 862 Б,
хотя `gh release view` уже показывал свежие 43 249 Б. Пара манифест+ядро при этом консистентно
*старая* — sha256-гейт проходит честно: владелец получает предыдущую ревизию, не битую (в отличие от
рассинхрона raw-канала выше). Урок: release-канал иммутабелен, только пока тег не перезаливали;
после clobber — подождать минуты или дождаться свежести манифеста (наш смок `smoke-release-15.mjs`
теперь ждёт совпадения sha256 манифеста с локальным dist перед прогоном).
