# Bug 04 — канал `main`: CDN-кэш raw.githubusercontent отдаёт рассинхронный набор артефактов

**Status:** 🔧 фикс отгружен (подсказка в загрузчике + докам; корень — природа канала)
**Version/build:** dist от d3a8f1e · **When/context:** 2026-07-16, полевой тест ДЗ-02 (homeworks/02),
слабая ЛЛМ в Zoo Code.

## Symptom

Загрузчик падает: `sha256 mismatch for KAIF-CORE-BUNDLE.md` при `--channel main`. Слабая модель честно
остановилась на гейте (и предложила владельцу неверные выходы: канал release, которого ещё нет, и обход
контрольных сумм).

## Repro (deterministic в окне рассинхрона)

Несколько пушей в main подряд → в окне кэша CDN запросить
`raw.githubusercontent.com/…/main/dist/kaif-manifest.json` и `…/main/dist/KAIF-CORE-BUNDLE.md`.

## Forensics (проверено в момент инцидента)

- manifest (raw main): version 1.4, bundle sha `0e81b2…` — СВЕЖИЙ (== локальному dist и репо);
- core (raw main): sha `bb7d9f…` — совпадает с манифестом — СВЕЖИЙ;
- bundle (raw main): sha `bbebb3…` — НЕ совпадает — СТАРЫЙ кэш (предыдущая сборка).
Репозиторий консистентен; рассинхрон только в раздаче CDN (кэш на каждый путь независим).

## Root cause

Канал `main` (raw по ветке) по природе mutable: CDN кэширует каждый файл независимо, после пуша набор
«манифест+ядро+бандл» может временно состоять из разных снимков. Загрузчик, обнаружив несовпадение,
корректно отказывается — **гейт сработал как задумано** (иначе слабой модели уехал бы битый набор).

## Fix

1. **Подсказка в ошибке загрузчика:** при sha256 mismatch теперь печатается готовое решение —
   иммутабельный `--source` c пином на коммит (`raw…/<full-sha>/dist`) и напоминание НИКОГДА не обходить
   проверку. Слабой модели больше не нужно гадать.
2. **Доки:** в `_thin-intro.md` (тонкий KAIF.md) и homeworks/02–03 канал `main` помечен как dev-канал с
   этим ограничением; продакшен-путь — релизный канал (ассеты иммутабельны, скью невозможен).
3. Обход не реализуем принципиально: bypass-флага у загрузчика нет и не будет.
