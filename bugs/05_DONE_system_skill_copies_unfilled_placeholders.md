# Bug 05 — копии навыков для агентских систем разворачивались с сырыми плейсхолдерами

**Status:** ✅ DONE
**Version/build:** dist 7d03198 · **When/context:** 2026-07-17, чистый прогон ДЗ-02 (homeworks/02),
слабая ЛЛМ в Zoo Code — второй настоящий баг, пойманный полевым тестом.

## Symptom

Канонические `.claude/skills/*` заполнены (`npm run build`, git-команда коммита), а копии тех же
навыков в `.roo/commands/`, `.agents/`, `.grok/`, `.cline/` — с сырыми `<BUILD_COMMAND>`/
`<COMMIT_COMMAND>`. Слабая модель принялась править ~100 файлов копий руками (задание «добей
плейсхолдеры — грепни каждый») и в `.agents/skills/autoloop/SKILL.md` вставила номера строк из гаттера
редактора (порча класса bug 01); попутно спотыкалась об `apply_diff` Zoo Code (требует 100% совпадения,
модель воспроизводила блок по памяти — 95%).

## Root cause

В `cmdInstall` заполнение плейсхолдеров шло в ЛОКАЛЬНУЮ переменную (`const content = fillPlaceholders(
f.content …)`) без записи обратно в `f.content`; `deployAgentSystems(skillFiles…)` выводил копии из
`deploy`-записей — то есть из СЫРОГО шаблона. Триггер-алиасы при этом в копиях БЫЛИ — их applyLanguage
мутирует прямо в записи (`entry.content = …`) — что и усыпило песочную проверку: копии проверялись на
алиасы, но не на плейсхолдеры (проверяли не то свойство — см. EXP-0010).

## Fix

`f.content = content` после заполнения/анонимизации в `cmdInstall` И в `cmdUpdate` — все производные
поверхности наследуют заполненный текст. Верификация: пересборка + песочная установка → греп
`<BUILD_COMMAND>|<COMMIT_COMMAND>` по `.roo/.agents/.grok/.cline` пуст (см. коммит).
