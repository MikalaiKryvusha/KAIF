# Bug 06 — update молча заменяет «усыновлённые» адаптации владельца (kept-файлы, совпавшие со снапшотом)

**Status:** ✅ DONE (2026-07-17) — фикс вшит прямо в 1.5, ассеты перезалиты в релиз v1.5
**Version/build:** релиз v1.5 (release-канал) · **When/context:** 2026-07-17, первый полевой релизный
`/kaif-update` на реальном проекте владельца (ndim, 1.4→1.5) — сразу после закрытия ДЗ-03.
**Серьёзность: высокая** — это нарушение главного инварианта уважительного обновления
(«ничего вашего не может быть перезаписано»), хоть и восстановимое через git.

## Symptom

Механический проход `update` классифицировал 18 RU-адаптированных навыков `.claude/skills/*` и
`PHILOSOPHY.md` проекта ndim как «untouched → replace» и молча заменил их английскими шаблонами 1.5.
Потеряны (до восстановления): проектные вставки NDim в resume/report-bug/kaif-remove/propose-idea/
what-next и локальное решение владельца «все документы — на русском». Агент проекта заметил по
`git diff`, восстановил из git и записал урок у себя (ndim EXP-0047). В сводке апдейта файлы честно
показаны как `replaced` — но списком без диффа, на что владелец легко не смотрит.

## Root cause

Критерий замены — `sha(текущий) == sha(снапшот из deploy-manifest) ⇒ «не тронут» ⇒ безопасно заменить`.
Он корректен только для путей, которые прошлый деплой **сам записал** (снапшот = шаблон). Для путей,
которые прошлый (legacy-)апдейт **сохранил как есть** (kept: у ndim это были 1.4-адаптации), снапшот
зафиксировал ВЛАДЕЛЬЧЕСКИЙ контент — и «не менялся со снапшота» вовсе не значит «не адаптирован».
Манифест хранит sha «что лежит на диске», но не хранит **происхождение**: кто это положил — шаблон
или владелец.

## Fix

`deploy-manifest.json` получил происхождение пути: список `kept: [...]` — пути, которые деплой
**усыновил** как есть (не записывал из шаблона). Классификация в `cmdUpdate`:

- путь записан прошлым деплоем и не менялся (sha == снапшоту, НЕ в kept) → replace;
- путь **kept** (усыновлён) — даже при совпадении со снапшотом → **diverged**: сохранён и выдан в
  `merge-diverged` пункт задания (мёрж новостей руками, язык/вставки владельца неприкосновенны);
- менялся владельцем → diverged (как раньше).

Сводка задания обновления получила подсказку «Sanity-check with git diff: replaced files must carry
NO owner edits» (агент ndim поймал потерю именно диффом).
