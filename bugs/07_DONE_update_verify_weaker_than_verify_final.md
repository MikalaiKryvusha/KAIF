# Bug 07 — update-verify не даёт гарантий verify-final (копии систем разъехались с каноном)

**Status:** ✅ DONE (2026-07-17) — фикс вшит прямо в 1.5 вместе с bugs/06, ассеты перезалиты в релиз
**Version/build:** релиз v1.5 · **When/context:** 2026-07-17, тот же полевой релизный `/kaif-update`
на ndim, что и bugs/06.

## Symptom

После завершения апдейта (update-verify exit 0, самоочистка) канонические `.claude/skills/*` в ndim —
RU-адаптированные (агент проекта восстановил их после bugs/06), а копии тех же навыков для остальных
систем (`.agents/`, `.roo/commands/`, `.grok/`, `.cline/`) — английские шаблоны из апдейта. Дерево
разъехалось: системы получают разные навыки, проектные вставки NDim живут только в каноне. Ни один
гейт этого не отметил.

## Root cause

`cmdUpdateVerify` — это только греп чекпоинтов + самоочистка. Все финальные гарантии установочного
`verify-final` (re-sync копий из канона — урок bugs/05/EXP-0010, скан плейсхолдеров, самолечение
маркера, скан анонимных утечек) на update-пути **не выполняются**. Гарантии оказались свойством
ПУТИ (install), а не ПРОДУКТА (развёрнутое дерево): любые правки владельца между механическим
проходом и update-verify (легитимные — мёрж diverged, восстановление после bugs/06) в копии не
доезжают.

## Fix

Оба верификатора сведены к одной последовательности `runFinalGates(taskFile, tag, verb)`: греп
чекпоинтов → скан плейсхолдеров → скан утечек (анонимный режим) → самолечение маркера из снапшота →
re-sync копий из канона → самоочистка (для анонимного — включая ядро/манифест/handles). update-verify
даёт гарантии verify-final дословно, тем же кодом, не копией. Заодно `cmdUpdate` пишет в
deploy-manifest снапшот маркера (раньше update-путь его терял — самолечение после апдейта было слепым).
