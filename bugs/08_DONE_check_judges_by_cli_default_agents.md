# Bug 08 — `check` до самоочистки судил по CLI-дефолту агентских систем, а не по маркеру

**Status:** ✅ DONE (2026-07-17)
**Version/build:** v1.5 (перезалитые ассеты) · **When/context:** пойман нашим же смоком после
перезаливки ассетов v1.5: установка `--agents claude-code,codex`, затем `check` без флагов —
80 «MISSING» по `.grok/.cline/.roo`, которых в этой установке быть не должно.

## Symptom

На суженной установке (не все пять систем) `check`, запущенный до самоочистки (живой бандл ещё в
`.kaif/install/`), требует артефакты ВСЕХ пяти систем и падает. После самоочистки тот же `check`
зелёный (ветка deploy-manifest хранит фактический список).

## Root cause

Ветка `check` с живым бандлом строила ожидаемый список артефактов из CLI-переменной `AGENTS`
(дефолт — все пять систем), игнорируя фактический маркер развёртывания. Тот же класс, что bugs/07:
ожидание зависело от пути вызова, а не от развёрнутого дерева.

## Fix

`check` наследует `agents` из `.kaif/kaif.json`, когда `--agents` не передан (как это всегда делал
`update`). [TESTED: 2026-07-17 · песочница A2: `check` до самоочистки на суженной установке
(claude-code, codex, zoo-code) — exit 0; полный прогон 27/27 зелёный].
