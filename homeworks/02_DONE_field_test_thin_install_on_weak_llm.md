## Подготовка (5 минут)

1. Создайте чистый тестовый проект (пустая папка + `git init`; можно с маленьким `package.json`).
2. Скачайте **тонкий** `KAIF.md` из репозитория: `dist/KAIF.md` (ветка main) — положите в корень
   тестового проекта.
3. Откройте проект в **Zoo Code** со слабой локальной моделью (та же, что в homework 01 — Ollama/LM
   Studio; контекст ≥ 8k желателен, но проверить хочется и на меньшем).

## Результаты (прогоны 1–5, 2026-07-16/17; хроника — агент, наблюдения — владелец)

**Стенд:** Zoo Code 3.68 (и Cline 4.0.8 в одном прогоне) · gemma-4-12b локально через llama-swap
(llama-server) · Windows 11 · рабочий язык ru.

**Вердикт: ЗАЧТЁН.** В прогоне №5 gemma-4-12b прошла путь целиком **без содержательной помощи**:
бутстрап (3 чекпоинта `KAIF-BOOT:`) → загрузчик (sha256 ok) → механическая установка (52 файла + 108
агент-артефактов, owner-доки ru, навыки EN с ru-алиасами) → все 7 чекпоинтов задания командой →
`verify-final` → самоочистка. Микро-сбои модели (кириллическая «с» в команде, галлюцинация пути
`.klaude`) самовосстановились по ошибкам инструментов. Финальный отчёт модели соответствует реальности.

| Критерий | Вердикт |
|---|---|
| 1. Три `KAIF-BOOT:` дословно | ✅ (зафиксированы в диагностиках/чате) |
| 2. Загрузчик exit 0, sha256 ok | ✅ |
| 3. Механика: ~52+108, ru/EN раскладка | ✅ |
| 4. `check` exit 0 | ✅ |
| 5. Ноль придуманных путей на диске (bug 01) | ✅ (манифест 100%; галлюцинации были только в read-попытках) |
| 6. Финал: verify-final, самоочистка, ядро живо | ✅ (маркер чинился вручную в №5 — теперь самолечится) |
| 7. Слабая модель дошла до конца | ✅ (2 смены харнесса — по вине харнесса, не KAIF) |

**Семь полевых находок → все механизированы (в этом и была цель теста):**
1. sha256 манифеста ≠ raw-раздаче (CRLF рабочей копии vs нормализация блоба; первый диагноз «кэш CDN»
   опровергнут контрфактом модели) → сборка строго LF + `dist/** -text` (bugs/04, EXP-0009).
2. Копии навыков систем уезжали с сырыми плейсхолдерами → write-back заполненного контента (bugs/05)
   + re-sync копий из канона в `verify-final`.
3. Слабые модели ломаются на **механике длинных tool-вызовов**, не на смысле: склейка `",path:` в
   content (Zoo run 3); Zoo `apply_diff` не чистит `NN |` в REPLACE → отравление файла (run 4; issue
   Zoo-Code-Org/Zoo-Code#910) → **команда `checkpoint`** вместо любого редактирования задания + правила
   в шапке задания.
4. Кириллическая «с» в shell-команде («сnode») — самовосстановление по CommandNotFound; правки не нужны.
5. Галлюцинация пути (`.klaude`) — самовосстановление по «file not found»; на диск не протекло.
6. Cline 4.0.8 + llama-swap: петля таймаутов (TTL 300s выгружает модель; prompt-eval > таймаута) —
   настройки харнесса (поднять Request Timeout и `ttl`), не дефект KAIF.
7. Модель перезаписала `.kaif/kaif.json` одним ключом → **команда `sphere`**, снапшот маркера в
   deploy-manifest, самолечение в `verify-final`.

**Закрывает `bugs/01`**: механическая распаковка на слабой локальной ЛЛМ подтверждена — ноль
интерпретаций структуры, всё по манифесту.
