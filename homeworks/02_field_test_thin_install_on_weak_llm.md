# Homework 02 — полевая проверка тонкой установки 1.5 на слабой локальной ЛЛМ (Zoo Code)

**Статус:** 🔴 ОТКРЫТО · выдано 2026-07-16 · Фаза 4 плана 13 (эпик 1.5 — Tested KAIF)
**Зачем:** главный тезис Thin KAIF — установка одинаково легка для сильной и слабой модели, потому что
всё делает машинерия. Проверить это можно только на реальной слабой локальной ЛЛМ — у агента её нет.
Успех закрывает и `bugs/01` (слабые ЛЛМ интерпретировали вместо копирования). Протокол заменяет
`homeworks/01` (тот был для установки 1.2).

---

## Подготовка (5 минут)

1. Создайте чистый тестовый проект (пустая папка + `git init`; можно с маленьким `package.json`).
2. Скачайте **тонкий** `KAIF.md` из репозитория: `dist/KAIF.md` (ветка main) — положите в корень
   тестового проекта.
3. Откройте проект в **Zoo Code** со слабой локальной моделью (та же, что в homework 01 — Ollama/LM
   Studio; контекст ≥ 8k желателен, но проверить хочется и на меньшем).

## Шаги теста

4. Скажите агенту (дословно, на русском — заодно проверим восприятие):
   > Прочти KAIF.md и распакуй фреймворк KAIF в этот проект. Рабочий язык: русский.
   >
   > ⚠️ ВАЖНО (только до релиза 1.5): при запуске загрузчика добавь флаг `--channel main`
   > (релизных артефактов ещё нет; после релиза 1.5 этот флаг будет не нужен).

   Если загрузчик упадёт с `sha256 mismatch` — это кэш-рассинхрон CDN канала main (bug 04), НЕ повод
   отключать проверки. Дайте модели закреплённый источник:
   `--source https://raw.githubusercontent.com/MikalaiKryvusha/KAIF/<полный-SHA-последнего-коммита-dist>/dist`.
5. **Не помогайте агенту.** Наблюдайте. Если застрял безнадёжно — зафиксируйте, где именно, и
   остановите.
6. Когда агент объявит, что механическая часть готова, — дайте ему доработать
   `KAIF_ADAPTATION_TASK.md` до конца (включая `node .kaif/kaif-core.mjs verify-final`).

## Что принести обратно (впишите прямо сюда, ниже)

- **Три строки `KAIF-BOOT:`** — напечатал ли агент их дословно (node-версия / loader written / exit 0)?
- **Строку итога установки** — `✅ KAIF … deployed mechanically (…)` с её цифрами.
- Вывод `node .kaif/kaif-core.mjs check` (запустите сами в терминале проекта — exit code и строку).
- Результат `verify-final` (прошёл/нет; если нет — какие отказы).
- `git status --short | wc -l` после установки (примерное число созданных файлов).
- Заметки: где агент спотыкался, что придумывал от себя (пути? имена?), сколько по времени/токенам.
- Название и размер модели, размер контекста.

## Критерии валидации (по ним агент судит результат)

| 2 | Загрузчик | exit 0; строки `sha256 ok` |
| 3 | Механика | `deployed mechanically` с ~52 файлами + 108 агент-артефактами; owner-доки по-русски, навыки EN с русскими алиасами |
| 4 | `check` | exit 0, «manifest satisfied» |
| 5 | Инвариант bug/01 | НОЛЬ придуманных путей/имён (всё легло туда, куда велит манифест — `check` это и ловит) |
| 6 | Финал | `verify-final passed`, KAIF.md/загрузчик/задание удалены, `.kaif/kaif-core.mjs` жив |
| 7 | Субъективно | Слабая модель дошла до конца без «утонула в контексте» |
