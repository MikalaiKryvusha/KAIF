# Idea 08 — Разведка конкурентов (EGC и аналоги) → видение по Парето 80/20

> Source: AI agent (по запросу владельца), 2026-07-07.
> Status: ❓ AWAITING HUMAN APPROVAL — идея агента = элемент видения; не реализуется без одобрения.

## Essence

Провести **лёгкую разведывательную работу** по конкурентам/родственным инструментам KAIF —
начиная с [EGC (Fmarzochi/EGC)](https://github.com/Fmarzochi/EGC) — и превратить её в **видение по
Парето 80/20**: те немногие шаги, которые дадут KAIF максимум приближения к уровню конкурентов.

Явное ограничение по объёму (указание владельца): **БЕЗ навыка глубокого исследования** (`/bug-research`
и любые токеноёмкие research-агенты не задействуются). Только ручной гуглёж + чтение репозиториев +
локальный анализ и рефлексия. Дёшево, целенаправленно, по KISS.

Работа состоит из трёх последовательных этапов; результат каждого — артефакт в репозитории:

1. **Найти и составить список.** Изучить EGC, погуглить аналоги (фреймворки/промпт-паки/скиллсеты,
   которые превращают AI-агента в дисциплинированного автономного напарника, устойчивого к потере
   контекста). Собрать список кандидатов с однострочным описанием и ссылкой.
2. **Локальное сравнение.** По каждому значимому конкуренту — что сделано **лучше, чем в KAIF** (чего
   KAIF не хватает на их фоне) и что сделано **хуже** (где KAIF выигрывает). Честно, конкретно, по
   пунктам.
3. **Видение по Парето 80/20.** На основе сравнения — короткий ранжированный список: те ~20% изменений
   в KAIF, что закрывают ~80% отставания. Каждый пункт → кандидат в отдельную идею (`/propose-idea`),
   которую владелец уже отдельно одобряет/отклоняет.

## Why / what problem it solves

KAIF развивается **изолированно** (дистилляция из личного опыта Криника + Claude). Нет карты
ландшафта: неизвестно, какие удачные ходы уже нашли соседи и где KAIF реально силён. Без этого легко
переизобретать чужое и не замечать собственных пробелов. Разведка + Парето-фильтр даёт владельцу
основанную на фактах повестку развития вместо догадок — при минимальной цене (без дорогого
research-контура).

## How it fits the master plan and vision

- **Цель проекта** — универсальный самораспаковывающийся фреймворк-напарник. Понимание конкурентов
  прямо питает `GOAL.md`/`MASTER_PLAN.md`: что должен уметь KAIF, чтобы быть выбором по умолчанию.
- **PHILOSOPHY (KISS + Оккам).** Парето-фильтр — это и есть бритва Оккама применительно к дорожной
  карте: не «сделать всё как у всех», а найти немногое, что даёт максимум.
- **Дисциплина видения.** Сама разведка — автономная и дешёвая (можно катать в `/dayloop`). Но её
  ВЫВОДЫ (что менять в продукте) — элемент видения, поэтому каждый Парето-пункт возвращается владельцу
  через `/propose-idea`, а не реализуется вслепую.
- Согласуется с `researches/` как местом хранения фактуры (ср. `researches/01_agent_skill_formats.md`).

## Implementation sketch (KISS)

- **Артефакт разведки** — `researches/02_competitor_landscape.md`: список инструментов (ссылка +
  1-строчное описание + к какому классу относится), затем таблица сравнения с KAIF по осям
  (память/устойчивость к контексту, автономность/циклы, распаковка/установка, мультиагентность,
  документная дисциплина, простота приёма, язык/локализация, лицензия/открытость). Колонки:
  «лучше нас / чего не хватает» и «хуже нас / где выигрываем».
- **Метод (без глубокого research-навыка):** README/доки/структура репозиториев, беглый гуглёж
  «AI agent framework / context engineering / autonomous coding agent memory», просмотр звёзд/активности
  как грубого сигнала зрелости. Свежесть источников проверять (экосистема быстро меняется).
- **Артефакт видения** — секция «Парето 80/20» в том же документе + пункты, разнесённые в отдельные
  `ideas/NN_*.md` (по одному на инициативу), каждый `❓ awaiting approval`.
- **Объём/риски:** пол-дня–день автономной работы. Риск — расползание в «глубокое исследование»
  (нарушит ограничение владельца) → держать строго на уровне README/сравнения, не закапываться.
