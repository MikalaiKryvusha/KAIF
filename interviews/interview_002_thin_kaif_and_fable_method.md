## Вопросы

### Q1. Как встраиваем fable-method?
- **А** — независимый модуль (плагин Claude Code + ссылки; для остальных 4 систем всё равно копирование руками).
- **Б** — «хирургически в ДНК»: дистилляция в собственные тексты KAIF (форк без обновлений, риск испортить
  выверенные формулировки парафразом).
- **В (рекомендую)** — гибрид: 4 навыка вендорим **дословно, EN, с шапкой-атрибуцией** (автор Sahir619,
  репозиторий, версия-пин v1.4.0, MIT) как шаблоны `framework/skills/fable-*`; в документы KAIF — только
  точки вызова и сшивка конфликтов; обновления — редким полуручным сравнением с апстримом.
  Обоснование: researches/03 §6.

### Q2. Объём вендоринга fable (если Q1 = В)
Базово беру: 4 SKILL.md + `failure-modes.md` + `flowcharts.md` + `examples.md` (~35–45K знаков,
`KAIF.md` вырастет ~на четверть; сейчас 175K). **Тянуть ли 8 доменных адаптеров (~ещё 30K)?** Они
пересекаются с нашими `framework/spheres/`: адаптеры богаче (evidence set, фроды, авторитеты сферы).
Варианты: не тянуть · тянуть и **заменить ими spheres** (рекомендую, если тянем) · держать оба (против KISS).

### Q3. Делать ли fable-judge обязательным шагом ритуалов?
Judge-проход перед «done» в `/dayloop`/`/nightloop` и перед публикацией в `/release`.
**Рекомендую: да** — это прямое закрытие Парето-пробела «надёжность исполнения» (researches/02), у KAIF
сейчас нет адверсарной верификации вообще. Цена — токены на проход судьи в циклах.

### Q4. Смена канона самораспаковки (важно!)
Сейчас канон (интервью 001 Q1, твоя заметка в AGENT_GUIDE): «всё необходимое живёт внутри `KAIF.md`».
Тонкий `KAIF.md` качает машинерию из origin → **без сети установка невозможна**. Предлагаю:
**канон меняется на «самодостаточен origin»**, а в каждый релиз дополнительно кладём тихий
оффлайн-ассет `KAIF-FULL.md` (сегодняшний полный формат, генерация уже есть) для сред без интернета.
Подтверждаешь?

### Q5. Политика установки Node
Загрузчик — mjs, ему нужен Node; проверить/поставить Node может только сам ИИ-агент до запуска загрузчика
(курица-и-яйцо, researches/04 §2.2). Предлагаю: **никогда молча** — если Node нет, агент спрашивает юзера
и ставит через пакетный менеджер платформы (winget/brew/apt) или даёт ссылку nodejs.org. Ок?

### Q6. Десять языков owner-шаблонов + триггер-алиасы
Предлагаю состав: **en, ru, zh-Hans, es, hi, ar, pt, fr, de, ja** (переводы готовятся один раз в репо
под ревью, не при установке; языки вне десятки — переводит ИИ на финальном шаге, как сейчас).
И: навыки остаются EN, но в `description:` машинерия дописывает **строку-алиас триггеров на языке
владельца** из микро-таблицы (иначе агент перестанет ловить команды на языке владельца —
конфликт с текущим правилом ядра, researches/04 §3.3). Подтверждаешь состав и алиасы?

### Q7. Судьба KAIF-SLIM
Идея 12: Thin KAIF так лёгок, что Slim не нужен. Предлагаю: **retire в 1.5** (файл и секции README
убираем; офлайн-случай закрывает `KAIF-FULL.md` из Q4; Slim-источник остаётся в git-истории).
Либо оставить Slim как «путь без Node/сети». Твоё слово.
