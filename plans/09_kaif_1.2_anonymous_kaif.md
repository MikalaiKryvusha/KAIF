## Шаги

### 1. Механическая распаковка ядра (фикс bug 01) — главный шаг
- [ ] Новый шаблон `framework/kaif-unpack.mjs` — маленький самодостаточный Node-скрипт-распаковщик:
      парсит блоки `FILE:` в `KAIF.md` и пишет файлы **дословно** по точным каноническим путям
      (существующие непустые файлы не трогает); режим `--check` — валидация манифеста (каждый путь
      существует и непуст, отчёт о недостающем, ненулевой код выхода при неполноте); режим
      `--agent zoo-code|roo-code` — механическая трансляция навыков в `.roo/commands/<name>.md`
      (отбросить `name:`, сохранить `description:` и тело) + `AGENTS.md` + `.roo/rules/kaif.md`.
- [ ] Встроить скрипт в ядро отдельным `FILE:`-блоком (свойство самораспаковки сохраняется: всё внутри
      `KAIF.md`); блоков становится **37** (9 доков + 6 README + 21 навык + 1 распаковщик).
- [ ] Переписать §8 ядра: этап 1 по умолчанию выполняется скриптом (для агента остаётся один крошечный
      механический шаг — записать блок скрипта на диск и запустить); ручной путь — фолбэк для сред без
      Node, со строгим манифестом и обязательным этапом 1.5 «ВАЛИДАЦИЯ → ДОДЕЛКА → повтор до 100%».
- [ ] Ужесточить договорённости: канонические имена файлов переименовывать ЗАПРЕЩЕНО;
      `KAIF_DEPLOYMENT_PLAN.md` — строгий пошаговый контракт, а не рекомендации.
