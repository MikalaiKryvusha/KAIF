## Цель

Дать KAIF **память об опыте** — один постоянно растущий документ `EXPERIENCE.md` («ОПЫТ»), куда агент
копит уроки («пробовали X → упало по R; пошли через Y → сработало»), сверяется с ним ПЕРЕД задачей и
ведёт его САМ. Плюс — усилить дисциплину «карты» (blast-radius) вместо графового движка. Фича — часть
полезной нагрузки (разворачивается в чужие проекты) и применяется к самому этому репозиторию (dogfooding).

Природа KAIF соблюдена: **plain markdown, без движков/векторов/БД**; поиск — grep и консольные утилиты.

## A. Документ `EXPERIENCE.md` (канонический ключевой документ)

**Имя файла — каноническое `EXPERIENCE.md`** (полезная нагрузка англоязычна; имена не переименовываются —
`bugs/01`). В прозе — «Experience (ОПЫТ)». Русская обвязка — свой `EXPERIENCE.md` на русском.

**Структура (grep-дружелюбная; теги — ИНЛАЙН на каждой записи, БЕЗ общего облака):**

> ⚠️ Уточнение владельца (2026-07-08): **общего облака тегов НЕ делаем.** Теги стоят рядом с каждым
> `EXP` — так `grep '#тег'` находит сами опыты напрямую, и нечему рассинхронизироваться.

```
# EXPERIENCE (ОПЫТ) — накопленный опыт ИИ-агента

> Растущий журнал уроков. Пиши КОРОТКО и греп-дружелюбно. Новые записи — сверху.
> Теги — инлайн на каждой записи. Поиск: grep '#loop' EXPERIENCE.md · grep '❌' -A4 · grep 'EXP-0042'

## Записи

### EXP-0003 · 2026-07-08 · ❌→✅ · #loop #context
**Ситуация:** <1 строка>
**Что делали:** <кратко>
**Итог:** ✅/❌ <кратко>
**Урок:** <главное, ради чего запись>   → <ссылка: bugs/NN, ideas/NN, plans/NN>
```

**Греп-якоря (договорённость формата):** стабильный id `EXP-NNNN`; ISO-дата; маркеры `✅`/`❌`/`❌→✅`;
инлайн-`#теги` на КАЖДОЙ записи. Тогда `grep '#loop'`, `grep '❌' -A4`, `grep 'EXP-0003'` — всё находится
одной командой. Дисциплина тегов: переиспользуй существующее имя тега (грепни перед изобретением нового).

**Граница с `bugs/`:** `bugs/` = дефекты (симптом/форензика/фикс). `EXPERIENCE.md` = уроки об уровне
подхода/развилок, в т.ч. УСПЕШНЫЕ; ссылается на баг, но не дублирует его.
