# План 11 — Опциональные хуки-энфорсеры несущих правил

> Источник: `ideas/10_optional_enforcement_hooks.md` (одобрена владельцем 2026-07-08),
> `researches/02_competitor_landscape.md` (§3, Парето-пункт 1). Дата: 2026-07-08.

## Цель

Дать KAIF **опциональный** слой механического энфорсмента для НЕМНОГИХ несущих правил — через хуки
хоста (где они есть), оформленный как расширение адаптеров в полезной нагрузке. Документы остаются
первичным носителем дисциплины; хуки делают критичные правила неотменяемыми там, где это дёшево. Бьёт
в корень `bugs/01` (правила-как-проза не исполняются слабыми ЛЛМ) и `bugs/02` (само-обрыв цикла).

## Несущие правила под энфорс (короткий список — не всё подряд)

1. **rebuild-after-edit** — правил `framework/*` → обязана пройти пересборка `KAIF.md` (иначе ядро рассинхронено).
2. **no-context-self-stop** — не обрывать автономный цикл по «переполнению контекста» (ровно `bugs/02`).
3. **status-before-pause** — обновить `STATUS.md` перед паузой/завершением.
4. **no-rename-on-deploy** — не переименовывать канонические имена файлов при распаковке (`bugs/01`).

## Шаги

- [ ] Раздел «Hooks (optional enforcement)» в шаблонах адаптеров `framework/adapters/<host>.md`:
      какой механизм хуков у хоста, какие правила и как проверяются. **Первым — Zoo Code** (приоритет №1),
      затем Claude Code (богаче hooks).
- [ ] Референс-набор хуков в полезной нагрузке (напр. `framework/hooks/<host>/` или блоки внутри
      адаптера): pre-pause проверка свежести `STATUS.md`; post-edit проверка «пересобрано ли ядро после
      правки `framework/*`»; guard-заметка против само-обрыва цикла (где хук не может — усиленная проза).
- [ ] Отразить в `framework/_intro.md` (секция адаптеров/развёртывания): энфорсмент **опционален**; при
      отсутствии хуков всё работает как раньше (на прозе). Пересобрать `KAIF.md`.
- [ ] Синхронно — русская обвязка (описание в `AGENT_GUIDE.md`, если уместно).
- [ ] Связать с планом 10: правило `no-context-self-stop` ссылается на фикс `bugs/02`; `rebuild-after-edit`
      — на инвариант «полезная нагрузка → генерируется ядро».

## Верификация

- [ ] `build-framework.mjs` зелёный; если добавлены FILE-блоки хуков — счётчик самопроверки обновлён.
- [ ] Референс-хуки синтаксически валидны для целевого хоста (Zoo Code/Claude Code — по офиц. докам,
      ср. `researches/01_agent_skill_formats.md`).
- [ ] Явно задокументировано: где хуков нет — падения назад на прозу, без поломки развёртывания.

## Развилки владельцу

- Порядок хостов: **Zoo Code → Claude Code** (по приоритету №1). Подтвердить финальный список несущих правил.
- Держать энфорс-список коротким (4 правила) — расширять только по мере доказанной боли (KISS).
