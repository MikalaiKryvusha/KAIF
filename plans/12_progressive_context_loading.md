# План 12 — Прогрессивная (ленивая) загрузка контекста: роутер

> Источник: `ideas/11_progressive_context_loading.md` (одобрена владельцем 2026-07-08),
> `researches/02_competitor_landscape.md` (§3, Парето-пункт 3). Дата: 2026-07-08.

## Цель

Ввести дисциплину **грузить только релевантный срез** руководящих документов на текущий шаг вместо чтения
целых документов «на всякий случай». Усиливает главный тезис KAIF (устойчивость к контексту) его же
средствами; развитие линии `KAIF-SLIM`. **Дисциплина/структура ссылок, не движок** — никаких векторов.

## Дизайн (KISS)

**Роутер контекста** — маленькая таблица «тип задачи → минимальный набор документов/секций». Живёт в
`AGENT_GUIDE.md` (не отдельный файл — экономим сущности, если не разрастётся). Пример:

```
| Тип задачи         | Читать (минимум)                                              |
|--------------------|---------------------------------------------------------------|
| Любая (обяз. мин.) | STATUS.md · PHILOSOPHY.md (кратко) · этот роутер · EXPERIENCE.md (греп по тегам) |
| Баг                | + BUG_FIXING_FRAMEWORK.md · bugs/<этот> · карта (blast radius) |
| Фича/идея          | + ideas/<эта> · MASTER_PLAN.md · релевантный план             |
| Правка шаблона     | + AGENT_GUIDE (слои) · инвариант пересборки                   |
| Развёртывание      | KAIF.md §8 · манифест распаковщика                            |
```

**Секционирование** ключевых документов якорями, чтобы адресовать срез, а не весь файл.

**Обязательный минимум** (не подлежит ленивости): `STATUS.md` + `PHILOSOPHY.md` (кратко) + роутер +
сверка с `EXPERIENCE.md` по тегам. Остальное — по типу задачи.

## Шаги

- [ ] Роутер-таблица в `framework/AGENT_GUIDE.md` (+ русская обвязка), рядом с чек-листом «перед задачей».
- [ ] Правило: «сначала прочитай роутер и обязательный минимум, затем только нужный срез, дочитывая по
      необходимости» — в чек-лист.
- [ ] Согласовать с планом 12-циклами: шаг 0 циклов и `/refresh-context` уважают роутер (не грузят всё).
- [ ] Проставить/проверить якоря секций в ключевых документах, на которые ссылается роутер.
- [ ] Пересобрать `KAIF.md`; проверить, что ссылки/якоря разрешаются.

## Верификация

- [ ] `build-framework.mjs` зелёный; ссылки роутера разрешаются.
- [ ] Роутер согласован с реальными именами/секциями документов (в т.ч. новым `EXPERIENCE.md`).
- [ ] Обязательный минимум явно включает `PHILOSOPHY` — чтобы «ленивость» не уронила дисциплину мышления.

## Развилки владельцу

- Роутер в `AGENT_GUIDE.md` vs отдельный `CONTEXT_ROUTER.md` — предлагаю **в AGENT_GUIDE** (KISS), вынести,
  только если разрастётся.
- Точный состав обязательного минимума — черновик выше; подтвердить.
