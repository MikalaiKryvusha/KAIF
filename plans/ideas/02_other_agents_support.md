# Идея 02 — полноценная поддержка агентов, отличных от Claude (`AGENTS.md`)

> Источник: ИИ-агент (дистилляция, v1.0), 2026-06-30.
> Статус: ❓ НА СОГЛАСОВАНИИ ВЛАДЕЛЬЦА — идея агента = элемент видения; не реализуется без согласования.

## Суть

Сделать фреймворк явно агент-агностичным: задокументировать, как подключать его для агентов, которые читают `AGENTS.md`
(или другие соглашения), а не `CLAUDE.md` + `.claude/skills/` из Claude Code.

## Зачем / какую проблему решает

`goal.md` гласит, что агентом может быть «Claude или любой другой агент». Сегодня навыки предполагают соглашение Claude Code
(`.claude/skills/<name>/SKILL.md`). Другие агенты обнаруживают команды иначе. Короткий
раздел соответствий делает фреймворк универсальным на практике, а не только на словах.

## Как вписывается в мастер-план и видение

Фаза 2 «Охват». *Содержимое* навыков уже агент-агностично; только обвязка обнаружения
специфична для Claude. Эта идея документирует варианты обвязки, не меняя сути полезной нагрузки.

## Набросок реализации (KISS)

- Добавить короткий подраздел «Адаптация под другие агенты» в `FRAMEWORK.md` (через `framework/_intro.md`):
  *содержимое* навыка переносимо; разместите каждый `SKILL.md` там, где ваш агент обнаруживает команды, или держите
  их как именованные документы, которые человек вызывает вручную.
- Генерировать/поддерживать `AGENTS.md` при развёртывании, который указывает на `AGENT_GUIDE.md`, рядом с `CLAUDE.md`.
