## 1. Видение

Есть некоторый проект, над которым работает разработчик. Он подключает ИИ-агента для кодинга (Claude или любого другого агента).
Он кладёт фреймворк KAIF в свой проект и просит агента распаковать и применить его. С этого
момента агент работает как дисциплинированный, автономный, устойчивый к потере контекста коллега: он никогда не начинает с
нуля, его можно оставить работать на часы, он эскалирует только те решения, которые действительно относятся к владельцу,
и оставляет после себя репозиторий, который документирует, как и почему построен проект.

Фреймворк — это *обобщённая, переиспользуемая дистилляция* рабочего метода, который органически
сложился между автором и Claude в их совместной работе: он выделен, абстрагирован от исходного проекта и
от автора и упакован так, чтобы подходить любому программному проекту.

## 2. Принципы

| Принцип | Что это значит на практике |
|-----------|---------------------|
| **Универсальность** | Работает для любого языка/стека/проекта. Никакой специфики исходного проекта или автора в полезной нагрузке. |
| **Самораспаковка** | Всё необходимое для развёртывания находится внутри одного документа — `FRAMEWORK.md`. |
| **Вынесенная память** | Состояние агента живёт в файлах (четыре руководящих документа), а не в чате. |
| **Ограниченная автономия** | Агент сам решает, что дёшево откатить; эскалирует бренд/UX/архитектуру. |
| **Простота** | KISS + бритва Оккама. Затык означает недопонимание, а не сложность. |
| **Фрактальность / dogfooding** | Фреймворк сам организует и оборачивает собственный проект. |
| **Один источник + многоязычность** | Ядро — единственный английский `FRAMEWORK.md`; многоязычность — при распаковке (агент разворачивает его в язык пользователя). README двуязычный; локальная обвязка — на русском. |
| **Локализованное развёртывание** | Исходники на английском; при распаковке агент спрашивает у разработчика предпочтительный рабочий язык и пишет развёрнутую обвязку на нём. |
| **Живые документы** | Агент сам поддерживает и со временем улучшает собственные руководящие документы. |

## 3. Результат

Ядро — это **`FRAMEWORK.md`** — один самораспаковывающийся документ, который (а) полностью описывает фреймворк
и (б) инструктирует агента распаковать структуру в любой проект. Он генерируется из `framework/`
(канонические шаблоны) с помощью `tools/build-framework.mjs`, поэтому шаблоны живут ровно в одном месте.

Фреймворк разворачивает в целевой проект следующую структуру:
- Четыре руководящих документа: `AGENT_GUIDE.md`, `PHILOSOPHY.md`, `BUG_FIXING_FRAMEWORK.md`, `STATUS.md`.
- Каталоги знаний: `plans/` (+`ideas/`), `bugs/`, `interviews/`.
- Двенадцать навыков в `.claude/skills/`.
- Соглашения: тег `DONE`, стиль коммитов, git-процесс, comment/narrate/fresh-logs/absolute-dates,
  build-your-own-harness, живые документы.

## 4. Разрешение фрактальности (важно)

Этот репозиторий одновременно *является* фреймворком и *обёрнут* им. Рекурсия разрешается единственным
правилом: **развёртывание в пользовательские проекты происходит только из `FRAMEWORK.md`**, никогда из
собственной dogfooding-обвязки этого репозитория (`AGENT_GUIDE.md`/`STATUS.md`/и т. д. в корне, которые посвящены разработке
фреймворка). См. `goal.md` §"Результатом работы" и `AGENT_GUIDE.md` → заметку о фрактальности.

## 5. Дорожная карта (фазы)

### Фаза 0 — Дистилляция ✅ (v1.0)
- Вычитать `goal.md`; изучить наработки совместной работы; выделить фреймворк.
- Написать универсальные шаблоны (`framework/`) и повествование (`framework/_intro.md`).
- Собрать `FRAMEWORK.md` через `tools/build-framework.mjs`.
- Применить dogfooding к репозиторию (корневая обвязка, `.claude/skills/`, `plans/`, `bugs/`, `interviews/`).
- README (EN+RU) + PDF, LICENSE (MIT), `.gitignore`.
- Опубликовать на GitHub; выпустить релиз v1.0.

**Веха:** `1.0` — универсальный, самораспаковывающийся, двуязычный, с dogfooding, выпущенный.

### Фаза 1 — Упрочнение (v1.x)
- Валидатор `tools/check-framework.mjs` (сбалансированные блоки `FILE:`, разрешимые ссылки), встроенный в сборку.
- Реальные испытания развёртывания в 2–3 разнородных проектах (CLI, сервер, библиотека); возврат полученных уроков.
- Подтянуть текст; расширить справку по плейсхолдерам; добавить короткий пример «прохода по развёртыванию».

**Веха:** `1.1`

### Фаза 2 — Охват (v2)
- Полноценные заметки по поддержке агентов, отличных от Claude (обвязка `AGENTS.md`, обобщённое обнаружение «команд»).
- Опциональные стартовые наборы под архетип проекта (веб-приложение, библиотека, конвейер данных).
- Вклады сообщества (issues/PR), руководство CONTRIBUTING.

**Веха:** `2.0`
