# Research 02 — Ландшафт конкурентов KAIF и видение по Парето 80/20

> Тема: кто ещё «дисциплинирует» ИИ-агента-программиста (память, устойчивость к контексту, автономность),
> что у них лучше/хуже KAIF, и какие ~20%改进 закроют ~80% отставания.
> Породивший документ: `ideas/08_competitor_research_and_pareto_vision.md` (одобрен владельцем 2026-07-08).
> Метод (по требованию владельца): **лёгкая разведка БЕЗ навыка глубокого исследования** — гуглёж + чтение
> README/докладов, без токеноёмких research-агентов. Живой справочник (DONE не помечается). Создан 2026-07-08.
>
> ⚠️ Уровни уверенности: **EGC** — прочитан репозиторий напрямую; остальные — по поисковым сводкам и
> обзорам (могли устареть/огрубиться). Точечная доверификация репозиториев — задача-хвост.

## 1. Список конкурентов и родственников

Класс А — «фреймворки дисциплины» (методология + документы/команды для агента), ближайшие к KAIF:

| Инструмент | Суть (1 строка) | Форма | Ссылка |
|---|---|---|---|
| **EGC** (Extended Global Context) | Локальный MCP-рантайм: постоянная память между сессиями, decision log, «уроки» с затуханием уверенности, failure-mining, guardian-хуки | npm + MCP-серверы | github.com/Fmarzochi/EGC |
| **BMAD Method** | Имитация agile-команды: 26 персон-агентов (PM/Architect/SM/Dev/QA); «документация — источник истины» (PRD/архитектура/истории) | Репозиторий команд/персон | github.com/bmad-code-org/BMAD-METHOD |
| **GitHub Spec Kit** | Спек-драйв: `/specify` → `/plan` → `/tasks`, структура поверх любого агента | Шаблоны + команды | github.com/github/spec-kit |
| **Claude Flow** | AI-native память: SQLite-персистентность, рой агентов, узел-«Queen» пишет знания проекта | Рантайм + SQLite | github.com/ruvnet/claude-flow |
| **SuperClaude** | Командный фреймворк для Claude Code: субагенты, экономия контекста (`/magic` и др.) | Пакет команд (Claude Code) | github.com/SuperClaude-Org |
| **Trellis** | Прогрессивная спецификация вместо «раздутого CLAUDE.md»: грузит только нужные стандарты/PRD/журналы сессий на текущий шаг | Спек-система | (spec-driven, Claude Code) |
| **Ralph** | Автономный цикл: крутится, пока все пункты PRD не закрыты | Loop-скрипт | (autonomous loop) |
| **SAM / fspec** | Автономный TDD: «PRD на входе → рабочее покрытое тестами приложение»; Gherkin Given/When/Then, каждая строка ↔ бизнес-правило | Спек+TDD система | (spec/TDD) |

Класс Б — «слои памяти» (переиспользуемы, но это компонент, а не фреймворк дисциплины):

| Инструмент | Суть | Форма |
|---|---|---|
| **agentmemory** (jayzeng / rohitg00) | Постоянная память для Claude Code/Codex/Cursor: факты, дневные логи, заметки, семантический поиск (qmd) | markdown + семантический индекс |
| **memsearch** (zilliztech) | Единый слой памяти, markdown + Milvus (векторный поиск) | markdown + вектор-БД |
| **OpenViking** (ByteDance) | Контекст-БД: память/ресурсы/навыки через файловую парадигму; иерархическая подача, самоперестройка по паттернам использования | Контекст-БД |

Класс В — «нативная память хостов» (базовый уровень, который KAIF не должен переизобретать):
CLAUDE.md / AGENTS.md / `.cursorrules`, Skills (`SKILL.md`), Cline **memory bank**, Cursor notepads,
Claude Code auto-`MEMORY.md` (агент сам дописывает уроки в сессии).

## 2. Сравнение с KAIF

**Природа KAIF (важно для честного сравнения):** документный, хост-агностичный, самораспаковывающийся
**один файл**; вынесенная память (`STATUS.md`); ограниченная автономия + эскалация видения; KISS+Оккам;
двуязычие; dogfooding. **НЕ рантайм**: ни MCP, ни векторной БД, ни хуков-энфорсеров.

### Что у конкурентов ЛУЧШЕ (чего KAIF не хватает)

1. **Энфорсмент через хуки харнесса — ГЛАВНЫЙ пробел.** EGC-guardian валидирует команды/записи и
   майнит провалы в уроки **«не полагаясь на кооперацию ИИ»** (хуки харнесса). У KAIF корректность держится
   исключительно на том, что модель *решит* следовать прозе — ровно слабое место `bugs/01` (слабые ЛЛМ не
   следуют) и `bugs/02` (агент сам обрывал цикл вопреки правилу). У KAIF энфорсмента ноль.
2. **Автозахват памяти и уроков.** EGC авто-инжектит контекст и авто-дистиллирует провалы в «уроки» с
   затуханием уверенности; agentmemory/Claude Flow — постоянная память с поиском. У KAIF память **ручная**
   (агент должен вспомнить обновить `STATUS.md`); систематического реестра уроков нет (это и есть пробел,
   на который нацелена `ideas/09_experience_memory_decision_log.md`).
3. **Спек-строгость и трассируемость.** Spec Kit / BMAD / SAM / fspec: PRD → план → задачи → тесты, каждая
   строка ↔ бизнес-правило, TDD. У KAIF есть `plans/ideas`, но нет трассы «требование↔тест» и TDD-дисциплины.
4. **Прогрессивная загрузка контекста.** Trellis/OpenViking грузят только релевантный срез на шаг. KAIF
   грузит целые руководящие документы (Slim смягчает, но ленивой подгрузки нет) — при том, что устойчивость
   к контексту — его же главный тезис.
5. **Структурные мультиагентные роли.** BMAD (26 персон), Claude Flow (рой + Queen). KAIF — одиночный агент
   + субагенты по случаю, без ролевой системы.
6. **Семантический поиск по памяти.** agentmemory (qmd) / memsearch (Milvus). KAIF — только grep.
7. **Наблюдаемость / синхронизация.** EGC: `egc watch`, дашборд токенов/вызовов, кросс-инструментальная
   синхронизация. У KAIF — ничего.
8. **Размер экосистемы.** EGC везёт 479 компонентов (63 агента/229 скиллов/76 команд/111 правил); BMAD
   популярен. У KAIF 21 навык.

### Что у KAIF ЛУЧШЕ (где выигрывает)

1. **Ноль зависимостей / ноль рантайма / ноль установки.** KAIF — чистый markdown: чтобы *быть*
   фреймворком, не нужен ни npm, ни MCP-сервер, ни SQLite, ни Node. Работает там, где вообще ничего нельзя
   установить; `KAIF-SLIM.md` — один файл. EGC/Claude Flow требуют рантайм и MCP/БД.
2. **Настоящая хост-агностичность на уровне документов.** Разворачивается в любого агента через
   `AGENTS.md`+трансляцию навыков, не привязан к MCP-хостам. (SuperClaude/BMAD-AT-CLAUDE — только Claude Code.)
3. **Радикальная простота как ЯВНЫЙ принцип (KISS+Оккам).** Большинство конкурентов *наращивают* машинерию
   (рои, БД, персоны). Тезис KAIF — минимализм; для малых проектов Slim по лёгкости вне конкуренции.
4. **Самораспаковывающийся единый артефакт.** Один `KAIF.md` = и инсталлятор, и описание. У конкурентов —
   репозиторий/пакет.
5. **Двуязычие + локализованное развёртывание** (обвязка на языке разработчика). Конкуренты — только англ.
6. **Ограниченная автономия + дисциплина эскалации** (`interviews/`): явное «что агент НЕ решает один».
   Конкуренты оптимизируют пропускную способность, а не это управление видением.
7. **Живые документы + доказательство dogfooding’ом** (фреймворк ведёт сам себя).
8. **Слой мышления** (`PHILOSOPHY.md`, `BUG_FIXING_FRAMEWORK.md`) — когнитивная дисциплина, а не только леса.

## 3. Видение по Парето 80/20

**Вывод:** отставание KAIF концентрируется в НАДЁЖНОСТИ ИСПОЛНЕНИЯ (правила-как-проза не исполняются) и
ПАМЯТИ ОБ ОПЫТЕ, а не в «мало фич». Значит, вкладываться надо туда, а не в рой агентов/векторные БД/дашборды.

**Витальные ~20% (закрывают ~80% отставания) — каждый пункт → отдельная идея на одобрение:**

1. **Опциональные хуки-энфорсеры (адаптер под хост).** Механически исполнять НЕМНОГИЕ несущие правила
   (пересобрать после правки шаблона; не обрывать цикл по «контексту»; обновить `STATUS` перед паузой) —
   там, где хост даёт хуки (Claude Code hooks и т.п.). Документы остаются первичны; хуки — там, где дёшево.
   Бьёт прямо в корень `bugs/01`/`bugs/02`. → **новая идея (10)**.
2. **Decision Log / память об опыте.** Подтверждено рынком (EGC, agentmemory, Claude Flow — у всех есть).
   Максимум ценности при минимуме цены, прямо ложится на принцип вынесенной памяти. → **уже
   `ideas/09_experience_memory_decision_log.md`** (эта разведка усиливает уверенность в ней).
3. **Прогрессивная (ленивая) загрузка контекста.** Усилить главный тезис KAIF его же средствами: грузить
   только релевантный срез руководящих документов на текущий шаг (в духе Trellis), развитие линии Slim.
   → **новая идея (11)**.

**Осознанно НЕ копируем (против природы/KISS/универсальности KAIF):** мультиагентные рои и 26 персон
(BMAD/Claude Flow), векторные БД памяти (Milvus/qmd — grep достаточно и стек-нейтрален), обязательный
TDD/спек-драйв (SAM/fspec — ломает универсальность), дашборды/телеметрию рантайма. Это «анти-Парето»:
много машинерии ради малого прироста для нашей ниши.
