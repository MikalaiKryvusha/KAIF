## Общая картина

В версию 1.5 влились **три идеи владельца**: №12 (тонкая установка + fable-method — ядро эпика),
№13 (культура тестирования — дала релизу имя **Tested KAIF**) и №14 (уважительное обновление старых
проектов). Работа шла по канону KAIF: глубокая проработка (researches/03, researches/04) → интервью с

Фазы 0–3 эпика и оба плана-спутника выполнены 2026-07-16 (8 коммитов в origin).

## 1. fable-method — дисциплина исполнения в ДНК KAIF (идея 12, часть 2)

**Что это.** [fable-method](https://github.com/Sahir619/fable-method) — дистилляция метода работы модели
Claude Fable 5: цикл «классифицируй запрос → определи "готово" → собери свидетельства → реши → действуй
хирургически → проверь наблюдением → доложи результатом-вперёд». Главная находка, доказанная их eval-ами:
**слабые модели следуют правилам на точках решения, а не правилам в списках** — принудительные
строки-артефакты (`INTENT:`, `AUTH:`, `TWINS:`, `PENDING:`) подняли слабую модель с 0/4 до 4/4. Это бьёт
ровно в Парето-пробел KAIF из разведки конкурентов (researches/02): «правила-как-проза не исполняются».

**Как встроили (выбор владельца — гибрид, Q1):**

- Четыре навыка (`/fable-method`, `/fable-loop`, `/fable-judge`, `/fable-domain`) вендорены **дословно,
  на английском**, с шапкой-атрибуцией (автор Sahir619, MIT, пин v1.4.0) и перечислимым списком
  минимальных патчей (домены→сферы KAIF, GSD→циклы KAIF, suite→апстрим). Формулировки апстрима не
  парафразились — их сила в отточенности (урок EXP-0006).
- В собственные документы KAIF вшиты **точки вызова**, не копии: чек-лист `AGENT_GUIDE` — пункт
  «исполняй по fable-циклу» + подсекция дисциплины; `BUG_FIXING_FRAMEWORK` — **intent gate** (перед
  правкой поведения: «код делает X, проверка ждёт Y, спека говорит Z» — расходятся → это и есть находка)
  и **twin check** (нашёл дефект — обыщи проект на близнецов, строка `TWINS:`).
- **Judge-проход обязателен** (Q3): циклы `/dayloop`/`/nightloop`/`/autoloop` не считают пункт беклога
  «готовым», пока адверсарный судья не перепрогнал заявленные проверки и не сверил дифф со скоупом;
  `/release` получил шаг 4.5 — вердикт REFUTED блокирует публикацию.
- Конфликты канонов сняты явно: развёрнутый KAIF = стоячая авторизация на рутинные коммиты (релизы,
  деплои, внешние публикации — по-прежнему только через `AUTH:` со словами владельца); уточнение задачи
  = один точечный вопрос в чате, интервью — для развилок видения.

**Сферы вместо доменов (Q2).** Адаптеры fable целиком не тянули — наши библиотеки сфер
(programming/science/design/business + шаблон) получили от них дисциплинарные секции: **обязательный
минимум свидетельств** (что вскрыть до действия), **порядок авторитетов** (чьё слово побеждает),
**верификация наблюдением** (что значит «проверено» в сфере), **таблица фродов** (по ней судья судит
некодовые работы) и **«готово» на примере**. Сферы теперь разворачиваются в проект (`.kaif/spheres/`).

## 2. Тонкая установка (идея 12, часть 1)

**Проблема.** Старый `KAIF.md` — 3495 строк: агент читал всё, переводил ~40 файлов, адаптировал вручную;
две развилки (полный/slim, быстрый/поэтапный) до начала работы. Карта установки показала 13 точек
когнитивной нагрузки, из которых неустранимы только 3: понимание проекта, цели и архитектуры
(researches/04).

**Решение — три слоя:**

| Слой | Размер | Что делает |
|---|---|---|
| Тонкий `KAIF.md` | **166 строк** (в 21 раз меньше) | Манифест + три бутстрап-шага с принудительными чекпоинтами `KAIF-BOOT:`: проверь Node (никогда молча не ставить — Q5) → запиши загрузчик дословно из FILE-блока → запусти с параметрами владельца |
| `KAIF-LOADER.mjs` | ~90 строк, ноль зависимостей | Качает машинерию из origin с проверкой sha256 по манифесту; подделка отвергается (проверено подменой байта); каналы: релиз (пин) / main / `--source` |
| `KAIF-CORE.mjs` | тяжёлая машинерия | Делает всё механизируемое; остаётся жить в `.kaif/kaif-core.mjs` как движок `kaif:*`-команд и обновлений |

**Что ядро делает без ума агента:** распаковывает 134-блочный бандл · **само заполняет плейсхолдеры**
из реалий проекта (имя — package.json, репозиторий — git remote, лицензия — LICENSE, build/test —
scripts, автор — git config) · раскатывает навыки **сразу под 5 систем** (Claude Code, Codex,
**Grok Build** — новый адаптер, верифицирован по веб-докам, Cline, Zoo Code) + AGENTS.md · весь wiring
(маркер `.kaif/kaif.json`, npm-хендлы, контекст-файлы) · языковые пакеты · валидация `check`.

**Агенту остаётся** `KAIF_ADAPTATION_TASK.md` — короткое задание на неустранимо-когнитивное (изучить
проект, заполнить карты, вывести план из цели, подтвердить сферу), каждый пункт с чекпоинтом
`KAIF-ADAPT:`; `verify-final` механически проверяет чекпоинты, ловит забытые плейсхолдеры и
самоочищается (KAIF.md, загрузчик, задание, временное — удаляются).

**Анонимность — теперь by design** (раньше — когнитивный скрабинг прозы): пропуск 4 origin-навыков,
механический вырез авторской заметки по маркерам `KAIF:AUTHOR-NOTE` в шаблоне, деэкспансия акронима,
финальный греп-гейт (установка не завершится при найденном имени автора или origin-URL), удаление
самого ядра с origin-адресом внутри.

**Смена канона (Q4):** «всё в одном файле» → **«самодостаточен origin»**; для сред без интернета в
каждый релиз кладётся полный оффлайн-`KAIF-FULL.md` (генератор — прежний). Slim уходит в 1.5 (Q7).
До релиза корневой `KAIF.md` не тронут — новые артефакты живут в `dist/` (тонкий KAIF.md ·
KAIF-CORE.mjs · KAIF-CORE-BUNDLE.md · kaif-manifest.json · KAIF-FULL.md), чтобы внешние инструкции не
сломались раньше Фазы 5.

## 3. Десять языков (Q6)

Переводились **только owner-документы** — то, что читает владелец: `GOAL.md`, `KAIF_FRAMEWORK.md` и
шесть README директорий знаний. **Все agent-документы и навыки — английские** (канон владельца). Чтобы
агент продолжал ловить команды на языке владельца, у каждого пакета есть микро-таблица
триггер-алиасов: машинерия дописывает в английский `description:` навыка строку локальных фраз
(«продолжи», «сделай релиз») — наследуется всеми 5 системами. Пакеты: ru, es, pt, fr, de, zh-Hans, ja,
hi, ar (+ en нативный) — по 9 файлов каждый, `framework/templates/languages/<код>/`. Язык вне десятки
деградирует честно: ставится английский + пункт перевода в задании агента. Русский пакет собран из
переводов нашей же обвязки (dogfooding окупился) — доступен владельцу на ревью.

## 4. Tested KAIF (идея 13 → план 14)

Новый **одиннадцатый ключевой документ `TESTING_FRAMEWORK.md`** (EN в payload, RU в корне): семь
канонических принципов тестирования ISTQB, развёрнутые в руководство для ИИ-агента, универсальное по
сферам — тестируемо всё, от функции до моста и мысли.

Главный механизм — **маркеры доверия**:

- всё нетривиальное, что агент генерирует, рождается с **`[NOT-TESTED]`** в своём комментарии — сырому
  доверять нельзя, LLM «думала, что верно» — не улика;
- проверка **наблюдением** (запустилось, отрисовалось, посчиталось — fable шаг 5) переводит маркер в
  **`[TESTED: дата · как проверено]`**;
- `[TESTED]` можно доверять, но толика сомнения остаётся всегда (принцип 1: баги есть всегда);
- тест нашёл дефект → `/report-bug` → фикс → перепроверка;
- **ложный `[TESTED]`** объявлен фродом — `/fable-judge` перепрогоняет маркеры как заявления (строка в
  таблице фродов сферы programming).

Маркеры — персистентная память о проверках, переживающая сессии: fable проверяет в моменте, маркер
сохраняет факт для будущих агентов и потомков. Обвязано всё: чек-листы, роутеры, BUG_FIXING, CLAUDE.md
(оба слоя). Релиз переименован: **`KAIF 1.5 — Tested KAIF`**.

## 5. Обновление старых проектов (идея 14 → план 15)

Опасение владельца «не представляю, как агент уважительно обновит 1.4 до 1.5» снято машинерией:

- **Слепки.** При установке ядро снимает sha256 всего записанного (`.kaif/deploy-manifest.json`).
- **`node .kaif/kaif-core.mjs update`** (или `npm run kaif:update`): качает свежую машинерию из origin
  (sha256-верификация), сравнивает диск со слепками — **нетронутое** заменяется новыми шаблонами (с
  автозаполнением), **правленое владельцем/агентом не перезаписывается никогда** и попадает в короткий
  `KAIF_UPDATE_TASK.md` вместе с новостями шаблонов («что изменилось в этой версии» — `templateNotes`
  в бандле); **контент владельца** (GOAL, STATUS, директории знаний) вне скоупа обновления вообще.
  Ядро меняет и само себя; язык/системы/сфера наследуются из `kaif.json`; `update-verify` проверяет
  чекпоинты `KAIF-UPDATE:` и самоочищается.
- **Legacy-путь для 1.4-проектов** (слепков нет): владелец кладёт свежий тонкий `KAIF.md` и просит
  «обнови KAIF» — инсталлятор сам детектит старый `kaif.json`, ничего существующего не трогает,
  добавляет новое (fable-навыки, TESTING_FRAMEWORK, сферы) и выдаёт update-задание вместо adaptation;
  поля маркера (sphere, tracking, язык) наследуются. После этого проект получает ядро и слепки — все
  последующие обновления механические.
- Навык `/kaif-update` переписан с «когнитивной миграции» на этот маршрут (оба слоя).

## 6. Как это проверялось — dogfooding культуры Tested KAIF

Каждое заявленное свойство проверено **наблюдением в песочницах**, не чтением кода: полный цикл
установки (52 файла + 108 артефактов систем) · негативные пути (verify-final с невыполненными
чекпоинтами — 18 точных отказов, exit 1; подделанный бандл — отказ по sha256) · анонимный цикл с
греп-сканом до и очисткой после · update/no-op/legacy · установки на ru/zh-Hans/ar/de · сохранность
пользовательского package.json · пост-очисточные `kaif:check`/`kaif:version`.

Именно наблюдение вскрыло **8 реальных багов**, невидимых в happy-path (все закрыты):

| # | Баг | Урок |
|---|---|---|
| 1 | Сборка падала на Windows-CRLF-чекауте (срезка комментария ждала LF) — `bugs/03` | Инструменты обязаны быть CRLF-терпимы; добавлен `.gitattributes` (LF всюду) |
| 2 | BOM в package.json → парс падал → инсталлятор **молча пересоздавал файл владельца** | BOM-терпимый парс; нечитаемый файл юзера никогда не перезаписывается (EXP-0007) |
| 3 | `check` парсил бандл, который `verify-final` удаляет | Постоянный deploy-manifest |
| 4 | Анонимная установка утекала автора (навык kaif-version, заметка шаблона, origin-URL в ядре) | Анонимность держится только механическим греп-гейтом |
| 5 | Невалидный JSON немецкой таблицы (ASCII-кавычка в строке) | Валидация всех таблиц узлом |
| 6 | `zh-Hans` не матчился (ядро лоуэркейсит `--lang`) | Регистронезависимый матч путей пакета |
| 7 | Валидатор legacy-режима требовал adaptation-задание | Валидатор требует задание своего режима |
| 8 | Копии навыков систем ехали без references | references путешествуют с навыком |

Уроки уровня подхода зафиксированы в `EXPERIENCE.md` (EXP-0005…0007).

## 7. Что осталось до релиза
